# UX Batch Implementation Plan ‚Äî COMPLETED

**Goal:** 8 UX improvements across filtering, navigation, transactions search, and a unified Settings page.

**Status:** All tasks implemented and TypeScript-clean.

## What Was Built

### Task 1 ‚Äî Date Apply in picker popup ‚úÖ
**File:** `src/client/components/FilterSidebar/FilterSidebar.tsx`

- RangePicker is controlled (`open` / `onOpenChange` state).
- `renderExtraFooter` renders an **Apply** button inside the popup ‚Äî clicking commits `pendingStart`/`pendingEnd` to the global filter and closes the picker.
- **Full time** and **Last year** quick-filter buttons remain in the sidebar below the picker (outside the popup).
- Pending dates sync from `filters.startDate/endDate` via `useEffect` so external resets (e.g. Reset Filters) are reflected.

### Task 2 ‚Äî On-type transaction search ‚úÖ
**File:** `src/client/components/TransactionTable/TransactionTable.tsx`

- Local `localSearch` state added.
- Typing filters `visibleRows` in-memory (case-insensitive description match) ‚Äî no API call.
- Pressing Enter or clicking the magnifier icon calls `onSearch(value)` ‚Üí fires real API fetch via FilterContext.
- Clearing the input also clears the API search.

### Task 3 ‚Äî Temp URL category filter in TransactionsPage ‚úÖ
**Files:** `src/client/hooks/useTransactions.ts`, `src/client/pages/TransactionsPage.tsx`

- `useTransactions(overrides?)` accepts `Partial<TransactionFilters>` merged on top of global filters.
- `TransactionsPage` reads `?categoryIds=<id>` from its own URL params (via `useSearchParams`).
- When present: a blue dismissable `<Tag>` shows "Category: [name]" at the top; closing it navigates to `/transactions` (removes the param).
- Category name is resolved from `allCategories` in FilterContext.

### Task 4 ‚Äî DRY category navigation ‚úÖ
**File:** `src/client/utils/navigation.ts`

```ts
export function txnsByCategoryUrl(categoryId: string): string {
  return `/transactions?categoryIds=${categoryId}`;
}
```

Used in both `AccountsPage` and `ReportsPage`. Clicking a category in either page navigates to the filtered transactions view.

### Task 5 ‚Äî Category quick-filter buttons ‚úÖ
**File:** `src/client/components/FilterSidebar/FilterSidebar.tsx`

Three `<Tag>` buttons above the category list:
- **All** ‚Üí `excludeCategories = []` (show everything)
- **None** ‚Üí `excludeCategories = [all ids]` (hide everything)
- **Preset** ‚Üí `excludeCategories = defaultExcludedIds` (restore DB defaults)

`defaultExcludedIds` flows: `FilterContext` ‚Üí `AppLayout.filterFormProps` ‚Üí `FilterForm` prop.

### Task 6 ‚Äî Account card last transaction date ‚úÖ
**File:** `src/client/components/AccountCard/AccountCard.tsx`

Renders `account.latestDate` (already present in `AccountSummary`) as "Last transaction: YYYY-MM-DD" at the bottom of each card.

### Task 7 ‚Äî Settings page ‚úÖ
**Files:** `src/client/pages/SettingsPage.tsx`, `src/client/pages/DatabasesPage.tsx`, `src/client/pages/ImportPage.tsx`, `src/client/components/NavBar/NavBar.tsx`, `src/client/App.tsx`, `src/client/components/AppLayout/AppLayout.tsx`

#### NavBar
- Removed Import and Config from the main `Menu`.
- Added a **Settings** `<Button>` at the right edge of the header (outside the Menu, uses `margin-left: auto` via `flex: 1` on the Menu).
- Button shows as `type="primary"` when on any `/settings/*` route.

#### DatabasesPage (`src/client/pages/DatabasesPage.tsx`)
- New page, renders `<DbPicker onSwitched={refreshAll} />`.
- `onSwitched` calls `refreshAll()` from FilterContext to update global state.

- Removed `<DbPicker>` ‚Äî DB management is now on the Databases page.
- On mount, fetches `databasesApi.list()` to find the active DB and shows it as a blue `<Tag icon={<DatabaseOutlined />}>` at the top.
